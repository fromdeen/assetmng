// DATA FLOW EXPLANATION
1. GPS devices send location data via HTTPS/MQTT/WebSocket with TLS encryption
2. Ingest Gateway validates and authenticates incoming requests, applies rate limiting
3. Stream Processing normalizes data and applies geospatial enrichment algorithms
4. Geofence Engine checks boundaries using PostGIS ST_Contains operations
5. Data stored in PostgreSQL with PostGIS extensions and TimescaleDB partitioning
6. Real-time events published to message bus for live updates
7. Vector tiles generated dynamically for map visualization
8. Frontend receives data via REST API and WebSocket connections
